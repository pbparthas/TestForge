💰 Cost Philosophy
✓
AI is Expensive

Every Claude API call costs real money. TestForge is designed to minimize unnecessary AI calls while maximizing value when AI is actually needed.

📈
Track Everything

Every AI call is logged with token counts, costs (USD/INR), agent type, and model used. Full visibility into where money goes.

🎯
Use AI Only When Necessary

Pre-checks using free/cheap methods filter out cases where AI isn't needed. AI is reserved for tasks that truly require intelligence.

💸 AI as Fallback: Duplicate Detection (AI Only When Needed)
Unlike the 12 agents above, duplicate detection uses a 3-tier cascade where AI is only called as a last resort after cheaper methods fail. This is the primary cost optimization strategy in TestForge.
How it works: When checking if a test case already exists, TestForge tries free/cheap methods first. AI is invoked only when simpler methods can't determine similarity (~10% of cases).
1 Hash Check
Cost: FREE
SHA-256 hash of normalized content (whitespace collapsed, case-insensitive)
Catches: Exact duplicates, copy-paste
~60% of duplicates caught here
→
2 Levenshtein
Cost: ~$0.0001
Edit distance algorithm for fuzzy matching (85% = duplicate, 60-85% = check with AI)
Catches: Typos, minor rewording, formatting differences
Flags duplicates OR passes candidates to AI tier
→
3 AI Semantic
Cost: ~$0.01-0.05
Claude Sonnet analyzes semantic meaning (only if Levenshtein found 60-85% candidates)
Catches: Semantically equivalent tests with completely different wording
Only called when Levenshtein is uncertain (max 5 comparisons)
📈
Result: ~90% cost reduction compared to sending every test case to AI for deduplication

Defense-in-Depth Security

TestForge implements 7 layers of security protection. Even if one layer is compromised, others continue to protect the system. All AI-generated code requires human review before production use.

1 Authentication
JWT Tokens Access (15m) + Refresh (7d)
bcrypt Password hashing, 10 salt rounds
Active Check Verify account not disabled
Rotation Refresh tokens rotated on use
2 Authorization (RBAC)
Roles admin, manager, lead, qae + custom
Permissions resource:action (testcase:create)
Teams Project-level role assignments
Protected System roles can't be deleted
3 Data Protection
AES-256-CBC Encrypt tokens, credentials
Random IV Unique per encryption op
Env Keys Keys from environment only
No Plaintext Passwords as hashes only
4 API Security
Rate Limit 100 req / 15 min per IP
CORS Whitelist frontend origins
Body Limit 10MB max JSON payload
Zod Schema validation on all inputs
5 AI Safety
Jailbreak 8 regex patterns block injection
Token Cap 4096 max output tokens
No Execute Code saved, never auto-run
JSON Only Structured output schemas
6 Human-in-the-Loop
Risk Score AI artifacts assessed first
Workflow Draft Review Approve
SLA Review deadlines + escalation
Feedback Rejections improve AI
7 Audit & Compliance
Full Logging Category, severity, metadata
Attribution User + IP on every action
Change Track Old/new values captured
Security Auth failures tracked
🛡 Prompt Injection Defense

AI systems are vulnerable to prompt injection attacks. TestForge implements regex-based detection blocking common jailbreak patterns.

🚫 Blocked Patterns (8 regex rules)
"ignore previous instructions" "you are now DAN/jailbroken" "pretend you have no restrictions" "system: override" "forget your rules" "bypass your safety" "act as if you have no guidelines" "disable your ethics"
Defense Action: Requests matching these patterns are immediately blocked, logged to the security audit trail, and the user is notified.
